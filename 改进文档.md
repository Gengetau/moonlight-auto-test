# 2. 当前发现的核心问题

------

# 问题一：直接访问 JSP 不符合真实系统行为

当前框架：

```
page.goto(
    ".../ProjectMemberUploadDisp.jsp"
)
```

但真实系统：

```
几乎不会直接访问 JSP
```

实际流程：

```
.do
  ↓
Action
  ↓
Forward
  ↓
JSP
```

例如：

```
ProjectMemberUpload.do
    ↓
ProjectMemberUploadDisp.jsp
```

因此：

```
直接 goto JSP
```

会导致：

- session/context 不完整
- frame 状态缺失
- parent/window.opener 缺失
- body 不 render
- popup about:blank

------

# 问题二：Legacy 页面白屏

当前现象：

| 环境   | 状态 |
| ------ | ---- |
| Legacy | 白屏 |
| New    | 正常 |

同时：

```
URL match = True
```

说明：

```
goto 已成功
```

但：

```
Legacy body 不可见
```

原因本质：

```
JSP 不是系统入口
```

而是：

```
Action Forward Result
```

------

# 问题三：popup about:blank

当前：

```
点击菜单
 ↓
新窗口打开
 ↓
about:blank
```

原因：

旧系统：

```
form.target = "WinUserUpload"
form.submit()
```

popup 依赖：

```
.do action
```

而不是：

```
直接 JSP
```

因此：

```
popup 被打开
但没有真正完成业务跳转
```

------

# 问题四：wait_until 不适合旧系统

当前：

```
wait_until="load"
```

或者：

```
networkidle
```

对于：

```
Struts
Frame
旧 JS
长期连接
```

容易：

```
永远等不到 complete
```

导致：

```
Timeout
```

------

# 3. 当前问题本质

当前 Moonlight 最大问题：

# 框架不知道：

```
“真正的业务入口是什么”
```

目前：

```
target_page = JSP
```

但实际上：

```
target_page ≠ entry_url
```

正确模型应该是：

```
target_page = 最终 view
entry_url  = 实际业务入口
```

------

# 4. 正确架构模型

推荐：

```
{
  "ProjectMemberUploadDisp.jsp": {
    "entry_url": "ProjectMemberUpload.do",
    "view_page": "ProjectMemberUploadDisp.jsp",
    "ready_selector": "input[name='uploadFile']"
  }
}
```

执行：

```
.do
 ↓
Action
 ↓
Forward
 ↓
JSP
```

而不是：

```
直接 JSP
```

------

# 5. 最终确认的重要结论

由于：

```
移行后地址栏依然是 .do
```

所以：

# 不需要解析 Spring Controller

只需要：

# 解析 Legacy 的 struts-config.xml

即可。

------

# 6. 推荐解决方案

# Priority 1（最高优先）

## 新增 Struts Route Resolver

解析：

```
struts-config.xml
```

提取：

```
<action path="/ProjectMemberUpload">
    <forward path="/ProjectMemberUploadDisp.jsp"/>
</action>
```

生成：

```
{
  "ProjectMemberUploadDisp.jsp": {
    "entry_url": "ProjectMemberUpload.do"
  }
}
```

------

# Priority 2

## 修改 RegressionEngine

当前：

```
goto(base_url + target_page)
```

改为：

```
goto(base_url + entry_url)
```

即：

```
legacy_url = legacy_base + route["entry_url"]
new_url = new_base + route["entry_url"]
```

------

# Priority 3

## 不再直接访问 JSP

JSP 只作为：

```
最终 view identifier
```

不作为：

```
浏览器入口 URL
```

------

# Priority 4

## wait_until 改造

统一：

```
wait_until="domcontentloaded"
```

并增加：

```
wait_for_selector()
```

等待：

```
业务元素
```

而不是：

```
浏览器 complete
```

------

# Priority 5

## 保留 Manual Mode

当前半人工模式是正确方向。

推荐流程：

```
1. 自动打开 TopMain
2. 人工登录/初始化 session
3. RegressionEngine 自动访问 .do
4. 自动等待业务元素
5. Screenshot / Diff
```

而不是：

```
人工点击 popup
```
