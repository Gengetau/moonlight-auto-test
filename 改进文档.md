------

# 3. 当前最可能根因

## 原因一：wait_until 等待策略不适用于旧系统

当前：

```
page.goto(url, wait_until="load")
```

或者：

```
wait_for_load_state("networkidle")
```

对于：

```
旧 Struts
旧 JS
Frame
Ajax
长期连接
```

很容易：

```
永远等不到 complete state
```

导致：

```
Timeout → 自动退出
```

------

# 4. 原因二：Frame/Parent 依赖

当前 JSP：

```
<html:form target="frMain">
```

说明系统使用：

```
frame / iframe 架构
```

旧系统可能依赖：

```
parent frame
window.opener
menu frame
header frame
```

因此：

```
直接打开 JSP
```

可能导致：

```
body 不渲染
```

------

# 5. 原因三：Legacy JS 报错

旧系统可能存在：

```
IE时代 JavaScript
```

在 Chromium 下：

```
JS error
资源加载失败
跨 frame 错误
```

导致：

```
页面初始化中断
```

------

# 6. 原因四：Session 虽然存在，但状态不完整

虽然：

```
URL 已进入 JSP
```

但：

```
业务上下文
菜单状态
frame context
```

可能未初始化。

旧系统经常依赖：

```
前一个菜单动作
```

才能正确 render 页面。

# 9. 改进方案

# Priority 1（最高优先）

## 修改页面等待策略

当前：

```
wait_until="load"
```

需要修改为：

```
wait_until="domcontentloaded"
```

原因：

```
旧系统可能永远等不到 load complete
```

------

## 推荐方案

```
page.goto(
    url,
    wait_until="domcontentloaded",
    timeout=60000
)

try:
    page.wait_for_selector(
        "form, table, input",
        timeout=15000
    )
except:
    pass

page.wait_for_timeout(2000)
```

核心思想：

```
不要等浏览器 complete
而是等业务元素出现
```

------

# Priority 2

## 增加 wait_page_stable()

新增：

```
wait_page_stable(page)
```

统一处理：

```
DOM ready
frame render
business element
延迟等待
```

------

# Priority 3

## 增加 Legacy Debug Layer

新增：

```
page.on("console")
page.on("pageerror")
page.on("requestfailed")
```

例如：

```
page.on(
    "console",
    lambda msg:
        print("[LEGACY CONSOLE]", msg.type, msg.text)
)

page.on(
    "pageerror",
    lambda err:
        print("[LEGACY JS ERROR]", err)
)

page.on(
    "requestfailed",
    lambda req:
        print("[LEGACY REQUEST FAILED]", req.url)
)
```

用于定位：

```
JS 崩溃
资源加载失败
frame error
```

------

# Priority 4

## 增强 Frame 自动检测

当前：

```
page.screenshot()
```

后续应：

```
自动识别有效业务 frame
```

例如：

```
for frame in page.frames:
    if frame.locator("body").is_visible():
        ...
```

或者：

```
检测包含 form/table/input 的 frame
```

------

# Priority 5


用于分析：

```
登录页
错误页
空白页
frame page
```
